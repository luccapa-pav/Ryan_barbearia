# Story 2.4 — cancelamento-flow

**Epic:** 2 — WhatsApp Agent
**Agente:** @dev
**Status:** Todo
**Branch:** feature/whatsapp-agent

## User Story

Como cliente, quero cancelar meu agendamento pelo WhatsApp de forma simples com confirmação, para que eu não precise entrar em contato manualmente com o barbeiro.

## Acceptance Criteria

- [ ] Busca agendamentos ativos (pendente|confirmado, futuros) do cliente por telefone
- [ ] Se nenhum agendamento: "Você não tem agendamentos ativos."
- [ ] Se um agendamento: mostra detalhes e pede confirmação
- [ ] Se múltiplos: lista numerada para escolha
- [ ] Confirmação: "Tem certeza que quer cancelar {serviço} em {data hora}? (sim/não)"
- [ ] Confirmação sim: UPDATE status='cancelado' + mensagem de confirmação
- [ ] Resposta não: "Ok, seu agendamento foi mantido."
- [ ] Fluxo dentro do limite de 2 turn (não precisa de sessão persistente)

## Tasks

- [ ] Criar query Supabase para buscar agendamentos ativos por telefone
- [ ] Criar lógica de seleção (1 vs múltiplos)
- [ ] Criar passo de confirmação com estado mínimo em sessoes_whatsapp
- [ ] Criar UPDATE de cancelamento
- [ ] Criar nó de envio WhatsApp para cada resposta
