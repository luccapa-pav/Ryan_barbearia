# Story 2.1 — whatsapp-receiver workflow + Evolution API

**Epic:** 2 — WhatsApp Agent
**Agente:** @dev
**Status:** Todo
**Branch:** feature/whatsapp-agent

## User Story

Como barbeiro, quero que o sistema receba mensagens do WhatsApp via Evolution API, classifique a intenção do cliente e roteie para o workflow correto, para que o atendimento seja automático desde a primeira mensagem.

## Acceptance Criteria

- [ ] Workflow `whatsapp-receiver` criado no n8n
- [ ] Webhook ativo no endpoint configurado pela Evolution API
- [ ] Deduplicação: mesma mensagem (messageId) não é processada duas vezes
- [ ] Normalização: extrai {telefone, texto, messageId, nome} independente do provider
- [ ] AI Classifier com GPT-4o-mini classifica em: agendar|cancelar|consultar|saudacao|outro
- [ ] Switch roteia para o sub-workflow correto
- [ ] Fallback para mensagens não reconhecidas
- [ ] Variáveis de ambiente configuradas no n8n

## Tasks

- [ ] Criar workflow `whatsapp-receiver` no n8n
- [ ] Configurar nó Webhook com path `/webhook/evolution`
- [ ] Criar nó de normalização (adapta payload Evolution API)
- [ ] Criar nó de deduplicação (check messageId)
- [ ] Criar nó AI Classifier (HTTP Request para OpenAI)
- [ ] Criar nó Switch por intent
- [ ] Criar nós de resposta para `saudacao` e `outro`
- [ ] Configurar credenciais no n8n
- [ ] Documentar JSON do workflow em docs/n8n/

```json
}
```
